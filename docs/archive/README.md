## 为什么归档

1. **认证改造主线已落地并稳定运行**，相关知识沉淀到 `docs/integration/` 下的现行文档：
   - `配置手册-环境变量.md` — 当前 `.env` 字段的权威说明
   - `认证体系总览.md` — 一份合并后的认证整体说明
2. **进入主线二：数据源改造**，避免历史文档与新方案在同一目录混淆。
3. 历史文档仍有参考价值（特别是排错思路与决策上下文），故保留在 `archive/`，不删除。
