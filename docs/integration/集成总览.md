# SQLBot-cx 数据中台集成总览

## 1. 目标

把 SQLBot-cx 集成进数据中台，让 SQLBot 作为"智能问数"模块复用中台的统一身份、统一数据源、统一权限。用户视角：**在数据中台登录后直接打开 SQLBot，无需再次登录、无需手动选数据源**。

## 2. 两条主线

### 主线一：认证改造（已落地，待收尾）

把 SQLBot 的认证从"只认本地 JWT"扩展为"识别数据中台 access_token（Redis 存储）"。

**核心思路：不复刻、只代理/透传**
- 登录：SQLBot 不自己签发中台 token，`/api/v1/login/access-token` 代理调用中台登录接口。
- 鉴权：SQLBot 中间件从请求头拿 Bearer token → 查中台 Redis `oauth2_access_token:<token>` → 恢复 `UserInfoDTO` 写入 `request.state.current_user`。
- 双通道：保留本地管理员白名单，确保开 strict 后 SQLBot 运维入口不失联。

详见 [`认证体系总览.md`](./认证体系总览.md)。

**状态：**
- ✅ 登录代理 / tenant-id 头处理 / Redis token 校验 / strict 模式 / 本地管理员双通道
- ⏳ refresh / logout 代理（未实现）
- ⏳ Redis miss → 中台 MySQL 回退（未实现，Redis 抖动即 401）
- ⏳ 中台部门/角色/scopes 到 SQLBot 权限体系的映射（当前只映射 id/account/tenantId）

### 主线二：数据源改造（进行中）

让用户在 chat 阶段**跳过数据源选择**，由中台 data 服务统一下发数据源与表。

**三段式目标：**
1. **跳过数据源选择**：chat/start 不再要求必填 `datasource`。
2. **远程拉取数据源**：在 `TokenMiddleware` 通过中台 token 后，注入虚拟 assistant（type=1），让 chat 主流程走 `AssistantOutDs.get_ds_from_api` 从中台 data 服务拉数据源。
3. **表级绑定 + 权限校验**：chat 可按 tableId 锁定单表；generate_sql 后调中台权限接口；无权则 SSE 返回申请流程。

详见 [`数据源改造方案.md`](./数据源改造方案.md)。

**状态：**
- 🚧 Phase 1：放宽 chat/start 必填 + 新增 `PLATFORM_DATASOURCE_*` 配置占位（开工中）
- 🚧 Phase 2：虚拟 assistant 注入（设计中）
- ⏳ Phase 3：表级绑定 + 权限 SSE

## 4. 核心决策（摘要）

| 决策 | 原因 |
|---|---|
| 不自建 OAuth2 发 token 服务，仅代理中台登录 | 中台已有完整 OAuth2 能力，重建风险高且违反"单一事实源"原则 |
| 中台 token 按随机字符串处理，不做 JWT 解码 | 中台 access token 不是 JWT；硬解码会触发 `Not enough segments` |
| Redis token 解析集中放 `token_resolver.py` | 多处鉴权（中间件/MCP/未来扩展）需复用，避免散乱 |
| strict 模式用于生产 | 兼容模式下中台随机 token 易被误入本地 JWT 解码分支 |
| 保留本地管理员双通道 | strict 开后仍需要 SQLBot 自身运维入口 |
| tenant-id 固定化而非每次查询 | 减少中台调用，降低延迟与依赖 |
| Redis DB 编号必须明确 | 历史踩坑：token 可能存在 db1，默认 db0 查不到 |
| **不复刻中台逻辑，只代理/透传** | 数据源/权限改造同样遵循此原则，新增配置用 `PLATFORM_*` 前缀 |

## 5. 已知风险

| 风险 | 影响 | 缓解 |
|---|---|---|
| Redis-only 校验，无 MySQL 回退 | Redis 抖动即全员 401 | 未来补 Redis miss → 中台 `check-token` 接口 |
| refresh/logout 未完整代理 | token 过期只能重登 | 列入 P1 待做 |
| `tenantId → oid` 粗映射 | 本地 ds 校验可能踩坑 | 数据源改造后统一走远程 ds 可绕过 |
| `isAdmin` 仅基于 `account=='admin' or user_id==1` | 不等同中台完整角色体系 | 主线二补角色映射 |
| `LOCAL_MODEL_PATH=/opt/sqlbot/models` 未随 `BASE_DIR` 改 | 部署时 embedding 可能失败 | 部署前核对 |
