# SQLBot-cx 中台账号密码登录接入实施说明

## 1. 目标

将 SQLBot 的 `/api/v1/login/access-token` 改为调用数据中台登录接口，使用中台账号密码换取中台 `accessToken`，并继续使用现有 `X-SQLBOT-TOKEN: Bearer <token>` 鉴权链路。

## 2. 已改造代码

1. 登录代理逻辑
- 文件：`backend/apps/system/api/login.py`
- 改造点：当 `PLATFORM_AUTH_LOGIN_PROXY_ENABLED=true` 时，`/login/access-token` 不再走本地 `authenticate`，而是请求中台登录接口。

2. 配置项
- 文件：`backend/common/core/config.py`
- 新增配置：
  - `PLATFORM_AUTH_BASE_URL`
  - `PLATFORM_AUTH_LOGIN_PATH`（默认 `/admin-api/system/auth/login`）
  - `PLATFORM_AUTH_REFRESH_PATH`（默认 `/admin-api/system/auth/refresh-token`）
  - `PLATFORM_AUTH_LOGOUT_PATH`（默认 `/admin-api/system/auth/logout`）
  - `PLATFORM_AUTH_LOGIN_PROXY_ENABLED`（是否启用登录代理）
- `PLATFORM_AUTH_FIXED_TENANT_NAME`（默认 `八院`）
- `PLATFORM_AUTH_FIXED_TENANT_ID`（例如 `124`，用于请求头 `tenant-id`）
- `PLATFORM_AUTH_TENANT_ID_BY_NAME_PATH`（默认 `/admin-api/system/tenant/get-id-by-name`）
- `PLATFORM_AUTH_TENANT_ID_HEADER`（默认 `tenant-id`）
- `PLATFORM_AUTH_REQUIRE_TENANT_ID_HEADER`（默认 `true`）
- `PLATFORM_AUTH_LOGIN_REMEMBER_ME`（默认 `true`）
- `PLATFORM_AUTH_HTTP_TIMEOUT_SECONDS`（默认 `10`）

## 3. 登录请求转发规则

SQLBot 接收前端的加密账号密码（沿用原逻辑解密），转发为：

```json
{
  "tenantName": "八院",
  "username": "<解密后的账号>",
  "password": "<解密后的密码>",
  "rememberMe": true
}
```

转发地址：

`{PLATFORM_AUTH_BASE_URL}{PLATFORM_AUTH_LOGIN_PATH}`

例如：

`http://100.66.219.51:8080/admin-api/system/auth/login`

中台成功返回后，SQLBot 返回：

```json
{
  "access_token": "<中台accessToken>",
  "token_type": "bearer",
  "platform_info": {
    "userId": 132,
    "refreshToken": "...",
    "expiresTime": 1775151217459
  }
}
```

前端无需改接口地址，仍调用 `/login/access-token`。

## 4. 与 Redis 鉴权改造的关系

本次是“登录入口”改造；你之前做的“Redis 恢复用户”在中间件继续生效：

- `backend/apps/system/middleware/auth.py`
- `backend/common/core/token_resolver.py`

建议配套开启：

- `PLATFORM_AUTH_ENABLED=true`
- `PLATFORM_AUTH_STRICT_MODE=true`
- `PLATFORM_AUTH_REDIS_URL=redis://127.0.0.1:6379/1`（按实际改）

这样所有 Bearer Token 都按中台 token 校验，不回退本地 JWT。

## 6. 联调验证步骤

1. 启动后端。
2. 用前端登录页输入账号密码（租户在后端固定为 `八院`）。
3. 前端拿到 `access_token` 后访问用户接口验证：

```bash
TOKEN='<中台accessToken>'
curl --noproxy '*' -i 'http://127.0.0.1:8013/api/v1/user/info' \
  -H "X-SQLBOT-TOKEN: Bearer ${TOKEN}"
```

预期：返回 `code=0`，且用户信息来自 Redis token 恢复。

## 7. 常见错误排查

1. 报错：`请求的租户标识未传递，请进行排查`
- 原因：中台要求 Header 中必须有 `tenant-id`，仅传 `tenantName` 不够。
- 处理：
  - 优先设置 `PLATFORM_AUTH_FIXED_TENANT_ID=124`
  - 同时确保 `PLATFORM_AUTH_TENANT_ID_HEADER=tenant-id`
  - 若不想写死，可启用 `PLATFORM_AUTH_TENANT_ID_BY_NAME_PATH` 自动按租户名查询 ID

## 9. 保留 SQLBot 管理员账号（双通道认证）

## 9.1 背景问题

当以下配置同时开启时：

- `PLATFORM_AUTH_ENABLED=true`
- `PLATFORM_AUTH_STRICT_MODE=true`
- `PLATFORM_AUTH_LOGIN_PROXY_ENABLED=true`

普通用户会全部走中台登录与中台 token 鉴权。此时如果不做额外处理，SQLBot 本地管理员账号（例如 `admin`）会失去登录入口。

## 9.2 解决策略

采用“双通道认证”：

1. 普通账号：走中台登录接口，返回中台 `accessToken`，后续请求走 Redis token 校验。
2. 管理员白名单账号：保留 SQLBot 本地账号密码登录，生成 SQLBot 本地 JWT，仅用于运维入口。

## 9.3 配置项

新增配置位于 `backend/common/core/config.py`：

- `PLATFORM_AUTH_ALLOW_LOCAL_ADMIN_LOGIN`：是否允许本地管理员登录
- `PLATFORM_AUTH_LOCAL_ADMIN_ACCOUNTS`：管理员账号白名单（逗号分隔）

示例 `.env`：

```env
PLATFORM_AUTH_ALLOW_LOCAL_ADMIN_LOGIN=true
PLATFORM_AUTH_LOCAL_ADMIN_ACCOUNTS=admin
```

说明：

1. 账号命中白名单时，登录接口优先走本地认证，不走中台登录代理。
2. 白名单匹配大小写不敏感，内部会转小写比较。
3. 支持多个管理员账号，例如 `admin,ops_admin,sa`。

## 9.4 代码实现位置

1. 登录分流逻辑：
- 文件：`backend/apps/system/api/login.py`
- 关键点：
  - `_is_local_admin_account(account)` 判断是否管理员白名单账号
  - `local_login()` 中先判断白名单，命中则执行 `_local_sqlbot_login(...)`
  - 非白名单再按 `PLATFORM_AUTH_LOGIN_PROXY_ENABLED` 决定是否调用中台

2. 鉴权分流逻辑：
- 文件：`backend/apps/system/middleware/auth.py`
- 关键点：
  - strict 模式优先校验中台 token（Redis）
  - 中台 token 失败时，尝试 `_try_local_admin_user(...)` 校验本地管理员 JWT
  - 非白名单本地 JWT 在 strict 模式下仍拒绝

## 9.5 行为矩阵

1. 白名单管理员账号登录成功：
- 返回 SQLBot 本地 JWT
- 后续请求可通过本地 JWT 校验

2. 普通账号登录成功：
- 返回中台 `accessToken`
- 后续请求通过 Redis `oauth2_access_token:<token>` 校验

3. 普通账号携带本地 JWT：
- strict 模式下会被拒绝（符合安全预期）

4. 白名单管理员携带中台 token：
- 也可通过（因为中台 token 校验在前）

## 9.6 验证步骤

1. 重启后端使 `.env` 生效。
2. 用白名单管理员账号登录一次，确认能拿到 SQLBot JWT（形如 `xxx.yyy.zzz` 三段）。
3. 用普通中台账号登录一次，确认拿到中台随机串 token（非 JWT 三段格式）。
4. 分别调用：

```bash
curl --noproxy '*' -i 'http://127.0.0.1:8013/api/v1/user/info' \
  -H "X-SQLBOT-TOKEN: Bearer <管理员JWT或中台token>"
```

预期：

1. 管理员 JWT：200
2. 中台 token：200
3. 非白名单本地 JWT：401（strict 模式）
